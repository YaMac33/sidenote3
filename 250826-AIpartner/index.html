25.8.27 AIへのプロジェクト質問方法

VS CodeやReplitでプロジェクトが大きくなってくると、ファイルが増えてきて「どのファイルを見せればいいんだろう？」「どうやって質問すればいいんだ？」と悩んでしまいますよね。
AIにプロジェクト全体を理解してもらい、的確なアドバイスをもらうには、いくつか効果的な方法があります。

結論から言うと：VS Code拡張機能の利用が最もおすすめ

現在の開発スタイルを大きく変えずに、最も効率的にAIのサポートを受ける方法は、VS Codeの拡張機能を使うことです。

  • GitHub Copilot Chat: 月額料金がかかりますが、最も強力な選択肢の一つです。チャット形式で「この変数どこで使ってる？」「このCSSを適用して」のように、開いているプロジェクト全体をAIが認識した上で対話できます。ファイルを一つ一つコピペする必要がありません。
  • Gemini (Google): Google公式の拡張機能です。選択したコードについて質問したり、新しいコードを生成したりできます。こちらもエディタ内で完結するので非常にスムーズです。
  • Cursor: VS Codeをベースにした「AIファースト」なエディタです。プロジェクト全体をAIが把握し、チャットで機能追加や修正を依頼できるため、非常に強力です。

これらのツールは、AIがあなたのプロジェクトの文脈（どのファイルがどこにあるかなど）を自動で理解してくれるため、質問の手間が劇的に減ります。

テキストベースで質問する場合のコツ

拡張機能を使わずに、ChatGPTやGeminiのWebサイトで質問する場合は、少し工夫が必要です。

1. treeコマンドでプロジェクト構成を伝える

まず、プロジェクト全体のファイル構成をAIに教えるのが効果的です。treeコマンドを使うと、ディレクトリ構造をテキストで簡単に表示できます。

使い方（VS Codeのターミナルで実行）：


# プロジェクトのルートディレクトリで以下のコマンドを実行
tree

もしtreeコマンドがインストールされていない場合は、お使いのOSに合わせてインストールしてください。これを実行すると、以下のようなテキストが出力されます。これをコピーして、プロンプトの最初に貼り付けましょう。


.
├── app.py         # Flaskサーバーのメインファイル
├── static
│   ├── script.js  # フロントエンドのJavaScript
│   └── style.css  # CSSスタイルシート
└── templates
    └── index.html # メインのHTMLファイル

このようにコメントで各ファイルの役割を書いてあげると、AIの理解度がさらに上がります。

2. 関連するファイルの内容だけを見せる

次に、質問に関連するファイルのコードだけを貼り付けます。すべてのファイルを貼り付けると、文字数制限に引っかかったり、AIが混乱したりする原因になります。

良い質問の例:

「以下の構成のWebアプリを開発しています。


.
├── app.py
└── templates
    └── index.html

index.htmlのフォームから送信されたデータを、app.pyで受け取ってコンソールに表示したいです。現在のコードは以下の通りですが、どこを修正すれば良いですか？

▼ app.py


from flask import Flask, render_template

app = Flask(__name__)

@app.route("/")
def index():
    return render_template("index.html")

if __name__ == "__main__":
    app.run(debug=True)

▼ templates/index.html


<!DOCTYPE html>
<html>
<head>
    <title>フォーム</title>
</head>
<body>
    <form>
        <input type="text" name="data">
        <button type="submit">送信</button>
    </form>
</body>
</html>

」

このように、「何がしたいのか」「現状はどうなっているのか」を明確に区切って示すのがコツです。
