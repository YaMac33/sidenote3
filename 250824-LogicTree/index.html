AIロジックツリー（マインドマップ）自動生成ツール 作成マニュアル

本ページでは、AIを活用したロジックツリー自動生成Webアプリの開発手順と技術選定の理由、公開環境の選び方を解説します。

1. どんな仕組みで実現するのか？

アプリは主に以下の3つで構成されます。

  • フロントエンド：ユーザー操作と結果表示（React + D3.js）
  • バックエンド：APIとAI連携（Python + FastAPI）
  • AI部分：関連語生成と関連度計算（Gemini APIやOpenAI API）

2. 開発ロードマップ

ステップ1：企画・設計

  • コア機能：入力→AIによる関連語生成→サイズ変化付き描画
  • 追加機能：保存、エクスポート、共有、デザイン変更など
要素目的技術選択肢推奨
フロント画面描画React / Vue.js / SvelteReact
描画ツリー表示D3.js / React Flow / GoJSD3.js
バックAPIサーバーPython(FastAPI/Flask), Node.jsPython+FastAPI
AI関連語生成Gemini API, OpenAIGemini API
公開デプロイVercel, Render, NetlifyVercel+Render

ステップ2：開発

バックエンド（FastAPI）例：

from fastapi import FastAPI import requests app = FastAPI() @app.post("/generate-nodes") def generate_nodes(keyword: str): prompt = f"{keyword} に関連する語を5つと関連度を0-1でJSON形式で出力してください。" # Gemini APIに問い合わせ（擬似例） res = requests.post("https://gemini.api/...", json={"prompt": prompt}) return res.json()

フロントエンド（React + D3.js）例：

import * as d3 from "d3"; // ノード描画の基本 function drawNodes(data) { const svg = d3.select("svg"); svg.selectAll("circle") .data(data) .join("circle") .attr("r", d => 10 + d.relevance * 50) .attr("cx", (d,i)=>100+i*100) .attr("cy", 200); }

ステップ3：テストと公開

  • 入力→ノード生成→サイズ変化が正しく動作するか確認
  • RenderにFastAPIを、VercelにReactをデプロイ

3. 技術選定の理由

Python + FastAPI の強み

  • AI APIのライブラリが充実
  • 少ないコードで高速に開発可能
  • 自動ドキュメント生成機能付き

React + D3.js の強み

  • 動的な状態管理が得意
  • D3.jsで関連度に応じた柔軟な図形描画が可能

4. 他の選択肢

  • フロントエンド：React Flow（ノード図特化）、Vue.js
  • バックエンド：Node.js（言語統一）

5. 費用を抑えられるサーバー

  • Vercel：Reactのフロント公開に最適（無料枠充実）
  • Render：FastAPIのバック公開に最適（無料DB付き）
  • Netlify：フロント向け。Vercelと同等
  • GitHub Pages：静的サイトのみ対応

おすすめ構成：フロントはVercel、バックはRenderで完全無料運用
