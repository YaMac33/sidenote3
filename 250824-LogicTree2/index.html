ロジックツリー自動生成Webアプリ - ファイル構成マニュアル

1. プロジェクト全体のフォルダ構成

「Python + FastAPI」のバックエンドと「React」のフロントエンドを分けて管理します。 

my-awesome-app/
├── .gitignore          # Gitで管理しないファイルを指定
├── backend/            # FastAPIのコード置き場
└── frontend/           # Reactのコード置き場
  • .gitignore：node_modulesや.venvなどを除外
  • backend/：サーバーサイド（FastAPI）のコード
  • frontend/：フロントエンド（React）のコード

2. バックエンド（Python + FastAPI）構成

backend/
├── .venv/               # 仮想環境（Git管理外）
├── .env                 # APIキーなど秘密情報（Git管理外）
├── main.py              # FastAPIアプリのメイン
└── requirements.txt     # 必要ライブラリ一覧

.env ファイル例

GEMINI_API_KEY="ここにあなたのAPIキーを貼り付け"

main.py の簡単な例

from fastapi import FastAPI
from dotenv import load_dotenv
import os

load_dotenv()  # .envを読み込む

app = FastAPI()
API_KEY = os.getenv("GEMINI_API_KEY")

@app.get("/")
def read_root():
    return {"Hello": "World"}

# ここにロジックツリー生成APIを追加

requirements.txt の例

fastapi
uvicorn
python-dotenv
google-generativeai

3. フロントエンド（React）構成

Viteを使ってReactプロジェクトを作成します：

npm create vite@latest frontend -- --template react
frontend/
├── node_modules/       # Git管理外
├── public/
├── src/
│   ├── components/
│   │   └── MindMap.jsx # D3.jsを使った描画処理
│   ├── App.jsx
│   └── main.jsx
├── .gitignore
├── index.html
├── package.json
└── vite.config.js

src/components/MindMap.jsx：D3.jsでロジックツリーを描画する主要コンポーネント。
src/App.jsx：全体レイアウトを制御。

4. iPadで開発するならReplitが最適

比較項目 Replit（推奨） ブラウザVS Code
コード編集◎◎
サーバー実行◎（FastAPI, React対応）×（不可）
ターミナル◎×
手軽さ◎△
公開機能◎（ボタン一つ）なし

結論：開発～実行～テストまで完結できるReplitが最適。
VS Code（ブラウザ版）はコード閲覧・軽微修正向き。
