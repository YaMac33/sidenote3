Googleフォームの回答を自動でNotionに保存する方法
（Google Apps Script編）

このマニュアルでは、Googleフォームに投稿された回答を、プログラミング（Google Apps Script）を利用して、自動的にNotionのデータベースへ保存する手順を解説します。

1. 準備するもの

スクリプトを設定する前に、以下の4つの項目を準備してください。

1-1. Notion APIキーの取得

Notionを外部から操作するための「APIキー（Internal Integration Token）」を取得します。

  1. Notionのインテグレーションページにアクセスします。
  2. 「+ New integration」をクリックし、名前（例: Google Form連携）を付けて作成します。
  3. 作成後、「Secrets」に表示される「Internal Integration Token」をコピーします。secret_...で始まる文字列です。

1-2. Notion データベースIDの取得

データを保存したいNotionデータベースの「ID」をコピーします。

  1. 対象のデータベースをフルページで開きます。
  2. ブラウザのアドレスバーに表示されるURLを確認します。
  3. URLのnotion.so/の直後にある、32文字の英数字がデータベースIDです。（例: https://www.notion.so/xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx?v=...）

1-3. データベースとインテグレーションの連携

作成したインテグレーションに、データベースを操作する権限を与えます。

  1. 対象のデータベースページの右上にある「・・・」メニューをクリックします。
  2. 「コネクトの追加 (Add connections)」を選択し、先ほど作成したインテグレーション（例: Google Form連携）を選んで連携させます。

1-4. フォームとデータベースの構成

このマニュアルのスクリプトは、以下の構成を前提としています。

Notion データベースの構成

プロパティ名 (列名) プロパティの種類
Prompt タイトル
Response リッチテキスト (または テキスト)
Timestamp 作成日時

Google フォームの構成

質問の種類 役割
記述式（短文） Notionの「Prompt」列に保存されます。
段落 Notionの「Response」列に保存されます。

フォーム内の質問の順番は問いません。スクリプトが自動で判別します。

2. 設定手順

準備が整ったら、Googleフォームにスクリプトを設定していきます。

2-1. スクリプトエディタを開く

  1. 対象のGoogleフォームの編集画面を開きます。
  2. 右上の「︙」（その他）メニューをクリックし、「スクリプトエディタ」を選択します。

2-2. スクリプトを貼り付ける

エディタが開いたら、以下のコードを貼り付けます。（元々書かれているコードは全て消してください）

// Notion APIにデータを送信する関数
function sendToNotion(e) {
  // ▼▼▼ ここにあなたのNotion情報を入力してください ▼▼▼
  const NOTION_API_KEY = "YOUR_NOTION_API_KEY"; // 1. あなたのNotion APIキー
  const NOTION_DATABASE_ID = "YOUR_DATABASE_ID"; // 2. あなたのデータベースID
  // ▲▲▲ ここまで ▲▲▲

  const formResponse = e.response;
  const itemResponses = formResponse.getItemResponses();

  let promptText = "";
  let responseText = "";

  // 全ての回答をループで確認し、質問タイプに応じて適切な変数に格納する
  for (let i = 0; i < itemResponses.length; i++) {
    const item = itemResponses[i].getItem();
    const response = itemResponses[i].getResponse();
    
    // 質問のタイプを判別
    switch (item.getType()) {
      case FormApp.ItemType.TEXT: // "記述式（短文）"の場合
        promptText = response;
        break;
      case FormApp.ItemType.PARAGRAPH_TEXT: // "段落"の場合
        responseText = response;
        break;
    }
  }
  
  // Notion APIのエンドポイントURL
  const url = "https://api.notion.com/v1/pages";

  // Notion APIに送信するデータ（ペイロード）を作成
  const payload = {
    "parent": { "database_id": NOTION_DATABASE_ID },
    "properties": {
      "Prompt": {
        "title": [{
          "text": { "content": promptText }
        }]
      },
      "Response": {
        "rich_text": [{
          "text": { "content": responseText }
        }]
      }
    }
  };

  // APIリクエストのオプションを設定
  const options = {
    "method": "post",
    "contentType": "application/json",
    "headers": {
      "Authorization": "Bearer " + NOTION_API_KEY,
      "Notion-Version": "2022-06-28"
    },
    "payload": JSON.stringify(payload)
  };

  // Notion APIにリクエストを送信
  try {
    const response = UrlFetchApp.fetch(url, options);
    console.log("Notionへの保存に成功しました: " + response.getContentText());
  } catch (error) {
    console.error("Notionへの保存中にエラーが発生しました: " + error.toString());
  }
}

2-3. APIキーとデータベースIDを書き換える

貼り付けたコードの以下の部分を、準備しておいたあなたの情報に書き換えます。

  • YOUR_NOTION_API_KEY: 準備1-1で取得したAPIキー
  • YOUR_DATABASE_ID: 準備1-2で取得したデータベースID

書き換えたら、フロッピーディスクのアイコン（プロジェクトを保存）をクリックして保存します。

2-4. トリガーを設定する（最重要）

このスクリプトが「フォーム送信時」に自動で実行されるように設定します。

  1. スクリプトエディタの左側メニューから、時計のアイコン（トリガー）をクリックします。
  2. 右下の「トリガーを追加」ボタンをクリックします。
  3. 以下の通りに設定し、「保存」をクリックします。
    • 実行する関数を選択: sendToNotion
    • イベントのソースを選択: フォームから
    • イベントの種類を選択: フォーム送信時

2-5. 権限を承認する

初めてトリガーを保存する際に、Googleアカウントによる承認を求められます。画面の指示に従い、スクリプトがフォームを操作することを許可してください。（「安全でないページ」と表示されても、詳細から許可へ進んでください）

3. 完了

以上で全ての設定は完了です。
今後、Googleフォームに回答が送信されるたびに、自動で指定したNotionデータベースに新しいページが作成されます。
