Googleフォームの回答を
Notionデータベースに自動保存する方法

はい、可能です。ただし、GoogleフォームとNotionは別のサービスなので、2つを連携させるための「仲介役」が必要になります。
直接的な標準機能はありませんが、以下のいずれかの方法を使えば、フォームに入力されたテキストを自動でNotionのデータベースに保存できます。

1. 外部連携ツールを使う方法（簡単・コード不要）

一番簡単でオススメなのが、Zapier (ザピアー) や Make (メイク) といった自動化ツールを使う方法です。プログラミングの知識は一切不要です。

仕組みのイメージ

「Googleフォームに新しい回答が送信されたら（トリガー）、Notionデータベースに新しいアイテムを追加する（アクション）」という命令を、画面上で設定するだけです。

🛠️ 代表的なツール

  • Zapier: 非常に有名で、設定画面が分かりやすいのが特徴です。無料プランでも月に100回までの連携が可能です。
  • Make (旧 Integromat): Zapierよりも複雑な連携ができます。無料プランの範囲も広く、Zapierで足りない場合に便利です。

設定の流れ（Zapierの場合）

  1. Zapierに登録し、「Create Zap（Zapを作成）」をクリックします。
  2. トリガーに「Google Forms」を選び、「New Form Response（新しい回答）」を選択します。
  3. アクションに「Notion」を選び、「Create Database Item（データベースアイテムの作成）」を選択します。
  4. 画面の指示に従い、GoogleアカウントとNotionアカウントを連携させます。
  5. 最後に、「フォームのどの回答」を「Notionのどの列」に保存するかを指定（マッピング）して完了です。

2. Google Apps Script を使う方法（無料・要コーディング）

Googleフォームに付属しているスクリプト機能を使えば、完全無料で同じことを実現できます。少しだけコードを書く必要がありますが、一度設定すれば非常に強力です。

仕組みのイメージ

フォームの回答が送信された瞬間に、裏側でプログラムが自動実行され、そのプログラムがNotion API（Notionを外部から操作するための機能）を叩いてデータを書き込みます。

設定の概要

  • Notionで準備:
    • インテグレーションを作成してインテグーショントークン（APIキー）を取得します。
    • データを保存したいNotionデータベースを、作成したインテグレーションに連携させます。
    • データベースID（URLに含まれる長い文字列）をコピーしておきます。
  • Googleフォームで準備:
    • フォームの編集画面から「スクリプトエディタ」を開きます。
    • 用意されたコードを貼り付け、取得したAPIキーとデータベースIDを書き込みます。
    • 「トリガー」を設定し、「フォーム送信時」にこのスクリプトが実行されるようにします。

✅ まとめ：どちらを選ぶべき？

それぞれの方法のメリット・デメリットを比較して、自分に合った方法を選びましょう。

方法 費用 難易度 おすすめな人
Zapier / Make 無料プランあり かんたん コードを書きたくない、すぐに連携したい人
Google Apps Script 完全無料 やや難しい コストをかけたくない、カスタマイズしたい人

まずはZapierで試してみて、もし利用頻度が多くなって有料プランが必要になったら、Google Apps Scriptを検討するのがおすすめです。
