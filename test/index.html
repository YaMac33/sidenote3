Windows (Codespaces) & Chromebook (Linux)
環境別・最適解のまとめ

【台本】 今回は、WindowsとChromebook、それぞれの環境でChrome拡張機能を開発する手順とポイントを解説します。 特に「環境構築が難しい」と感じている初心者の方に向けた、最も効率的な方法をまとめました。

開発における「最大の壁」

  • クラウド(Codespaces)の仕様
    ブラウザの中に隔離された環境
  • 拡張機能のセキュリティ制約
    「PC本体(ローカル)」のファイルしか読み込めない
  • 結論：ファイルの「転送」が必須
[イラスト] 雲(Cloud)からPCへ
矢印が伸びている図
「壁」があるイメージ
【台本】 GitHub Codespacesは便利ですが、拡張機能開発では一つ大きな壁があります。 それは、Chromeブラウザのセキュリティ仕様により、クラウド上のファイルを直接読み込めないことです。 そのため、どうしてもファイルを「PC本体」に持ってくるというひと手間が必要になります。

Windowsでの開発フロー

[イラスト] 実装 → ZIP転送 → 実行
3ステップのフローチャート
  • Step 1：実装 (Cloud)
    Codespacesで `manifest.json` 等を作成
  • Step 2：転送 (Download)
    ZIPでダウンロード、またはGitでPCへ同期
  • Step 3：実行 (Local)
    Chromeの「デベロッパーモード」で読み込む
【台本】 Windows環境でCodespacesを使う場合の具体的な手順です。 まずクラウドでコードを書き、それをZIPまたはGit経由でPCに保存します。 最後に、Chromeの設定画面から、保存したフォルダを読み込ませて動作確認を行います。

Chromebookでも開発可能？

[イラスト] Chromebookで
「OK！」サインを出すキャラクター
  • 答え：YES！（完全対応）
  • よくある誤解
    iPad等の「モバイルアプリ版」と混同されがち
  • 実際の中身
    Windowsと同じ「PC版ブラウザ」を搭載
【台本】 「Chromebookでは開発できない」と思っていませんか？それは大きな誤解です。 Chromebookに搭載されているのは、Windowsと同じ「PC版のChromeブラウザ」です。 そのため、デベロッパーモードを含むすべての機能が問題なく利用できます。

Chromebookの切り札
「Linux開発環境」

  • PCの中に「別のPC」を作る機能
    サンドボックス環境で安全に動作
  • プロ用ツールが動く
    PC版 VS Code, Git, Dockerなどが使用可能
  • 導入は簡単
    設定 → デベロッパー → オンにするだけ
[イラスト] Chromebookの中に
ペンギン(Linux)が住んでいるイメージ
【台本】 さらにChromebookには「Linux開発環境」という強力な武器があります。 これを使えば、マシンの内側に、プロのエンジニアが使うのと同等の本格的なプログラミング環境を構築できます。 設定からスイッチ一つでオンにできる手軽さも魅力です。

なぜLinux環境が最強なのか

  • 「ダウンロード」の手間ゼロ
    Linuxフォルダ内のコードを直接ブラウザで読める
  • 修正サイクルが爆速
    コード保存 → ブラウザ更新だけで反映
  • オフライン開発もOK
[イラスト] ストレスなく
サクサク開発している様子
（ダウンロード不要！）
【台本】 Linux環境を使う最大のメリットは、「ファイルのダウンロード」が不要になることです。 Linuxフォルダ内で書いたコードを、Chromeブラウザが直接読み込めます。 保存して更新するだけ。WindowsやMacでの開発と同じ、快適な開発サイクルが手に入ります。
