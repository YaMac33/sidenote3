25.8.27 WebアプリでテキストをHTML化し自動Pushする方法

概要

本文をHTMLに変換し、GitHubへ自動Pushするツールは十分に実現可能です。多くのブログやドキュメントサイトが似た仕組みを採用しています。

全体の流れ

  • 入力: 本文を貼り付けるフォーム（Webページ）
  • 処理: テキストをHTMLに変換
  • 出力: HTMLファイルをGitHubに自動Push

Webアプリ版の作り方

処理の流れ

  • フロントエンド (ブラウザ)
    • Webページを開く
    • テキストエリアに本文を貼り付け、「公開」ボタンを押す
    • 入力テキストをバックエンドへ送信
  • バックエンド (サーバー)
    • テキストを受信
    • Markdown → HTML変換（markdown-itなどを利用）
    • GitHub APIでHTMLファイルをPush（保存先やブランチを指定）
  • GitHub
    • Pushによりリポジトリ更新
    • GitHub Pages有効時は自動で公開

サーバー側のプログラム例（Python / Flask）

from flask import Flask, request, jsonify
import markdown  # MarkdownをHTMLに変換するライブラリ
import requests  # GitHub APIを叩くためのライブラリ
import base64
import os

app = Flask(__name__)

# --- GitHubの設定 ---
GITHUB_TOKEN = os.environ.get("GITHUB_PERSONAL_ACCESS_TOKEN") # 環境変数からトークンを読み込む
REPO_OWNER = "your-github-username"
REPO_NAME = "your-repo-name"
FILE_PATH = "public/new-page.html" # 保存したいファイルのパス
BRANCH = "main"

API_URL = f"https://api.github.com/repos/{REPO_OWNER}/{REPO_NAME}/contents/{FILE_PATH}"

@app.route("/publish", methods=["POST"])
def publish_to_github():
    text_content = request.json.get("text")
    if not text_content:
        return jsonify({"error": "テキストがありません"}), 400

    html_content = markdown.markdown(text_content)

    headers = {
        "Authorization": f"token {GITHUB_TOKEN}",
        "Accept": "application/vnd.github.v3+json",
    }
    encoded_content = base64.b64encode(html_content.encode('utf-8')).decode('utf-8')

    data = {
        "message": "新しいページを自動で追加",
        "content": encoded_content,
        "branch": BRANCH,
    }

    response = requests.put(API_URL, headers=headers, json=data)

    if response.status_code in [200, 201]:
        return jsonify({"message": "GitHubへのPushに成功しました！"}), 200
    else:
        return jsonify({"error": "GitHubへのPushに失敗しました", "details": response.json()}), 500

if __name__ == "__main__":
    app.run(debug=True)

このコードのポイント

  • /publish にPOSTすると実行
  • markdownライブラリで簡単にHTML変換
  • GitHub APIをPersonal Access Tokenで認証

必要な知識とツール

  • プログラミング言語（Python, Node.js など）
  • Webフレームワーク（Flask, FastAPI, Expressなど）
  • GitHub API操作（ファイル作成・更新）
  • HTML/CSS/JavaScript（フロントエンド作成）
  • サーバー環境（Render, Herokuなど）

まとめ

既存の技術を組み合わせれば実現可能です。まずは手元PCでHTML変換＋手動Pushのスクリプトを作り、全体像を把握することから始めるのがおすすめです。
